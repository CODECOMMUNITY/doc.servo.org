fn handle_exit_window_msg(&self, _: PipelineId)

We have gotten a window.close from script, which we pass on to the compositor. We do not shut down the script thread now, because the compositor will ask the constellation to shut down the pipeline, which will clean everything up normally. If we do exit, we will tear down the DOM nodes, possibly at a point where layout is still accessing them.
